Il problema

Il tuo AI sa cos'è un accordion.
Non sa quello che c'è adesso in Bootstrap Italia.

Filo è un server MCP non ufficiale che espone agli assistenti AI i dati strutturati del Design System .italia: markup HTML dei componenti Bootstrap Italia, web component Dev Kit Italia, token CSS con catena di risoluzione completa, linee guida Designers Italia, stato di accessibilità verificato e issue GitHub aperte. Dati aggiornati ogni notte dal branch data-fetched.

Ispirato a dati-semantic-mcp di Matteo Fortini — progetto analogo per schema.gov.it.

Setup

Installazione

NPX è il canale primario — nessuna installazione permanente. Docker per deploy stabili.

Avvio istantaneo via stdio — canale principale per Claude Desktop, Cursor, VS Code.

npx @fupete/design-system-italia-mcp
ℹ️
Su macOS con nvm, se npx risolve una versione vecchia di Node ("You must supply a command"), usa il path esplicito: nvm use 22 && which npx. Vedi la configurazione JSON nella sezione Configurazione.

Consigliato per deploy stabile. Supporta trasporto HTTP oltre a stdio.

docker pull ghcr.io/fupete/design-system-italia-mcp
docker run -p 8080:8080 \
  -e GITHUB_TOKEN=your_token \
  ghcr.io/fupete/design-system-italia-mcp
⚠️
Multiarch — se docker pull scarica un'architettura incompatibile, fai build locale: docker build -t filo-mcp .

Per sviluppo locale o per contribuire al progetto.

git clone https://github.com/fupete/design-system-italia-mcp.git
cd design-system-italia-mcp
npm install
cp .env.example .env
# aggiungi GITHUB_TOKEN in .env
npm run dev

Server su http://localhost:8080/mcp. Test: npx @modelcontextprotocol/inspector http://localhost:8080/mcp

ℹ️
GITHUB_TOKEN (opzionale ma consigliato) — serve per get_component_issues. Senza: 60 req/ora per IP. Con token read-only pubblico: 5000/ora.

Casi d'uso reali

Esempi d'uso

Esempi di query che usano i dati di Filo come base per la risposta dell'assistente AI.

"Dammi i token CSS per personalizzare il colore di header e footer. Sto usando Bootstrap Italia v3 con Jekyll."

→ get_component_tokens("header") + get_component_tokens("footer") + find_token("navbar") — token --bsi-* con catena di risoluzione fino al valore concreto

"Trova i componenti form disponibili e il loro markup Dev Kit Italia."

→ search_components("form") + get_component("input") — markup <it-input> estratto da Storybook

"Confronta i token di header e navbar — sono separati o condivisi?"

→ usa get_component_tokens("header") + find_token("navbar") — se i token navbar non sono presenti in get_component_tokens, il system prompt consigliato istruisce il modello a segnalarlo

"Quali componenti hanno problemi di accessibilità aperti?"

→ usa list_accessibility_issues — lista da dati verificati, non da conoscenza pregressa del modello

Una cosa da tenere a mente

Filo è un server che fornisce dati — markup HTML, token CSS, linee guida, stato accessibilità, issue aperte. È l'assistente AI che li usa per rispondere. I dati vengono da sorgenti aggiornate ogni notte, non dalla memoria del modello — ma il sistema prompt consigliato aiuta a mantenere questo confine.

Filo non fornisce calcoli WCAG, compatibilità browser, performance o esempi per React e Vue. Se l'assistente ti dà valori numerici precisi su questi temi, sono inferenze sue — non dati di Filo.

Anti-hallucination

System prompt consigliato

Il problema non è il server — è che i modelli mescolano dati MCP reali e conoscenza pregressa senza segnalarlo. Questo prompt rende visibile il confine e riduce le allucinazioni, anche con modelli piccoli.

## Data rules — Design System .italia (Filo MCP)

VERIFIED DATA: use exclusively the data returned by Filo's MCP tools. Do not supplement with prior knowledge of Bootstrap Italia, Dev Kit Italia, Design Tokens Italia, or any other CSS/web component framework.

WHEN DATA IS MISSING: if information is not present in the MCP responses, say so explicitly. Write "This data is not available in the MCP sources" instead of providing an estimate, plausible value, or inference. Never invent numeric values (WCAG contrast ratios, spacing, dimensions, colors).

WHEN COMPOSING ELEMENTS: if you combine real MCP markup with HTML/CSS structure you add yourself, clearly indicate what comes from MCP data and what is your own inference.

VERSIONS AND SOURCES: in every response that uses MCP data, include the source versions and official documentation URL returned by the tool.

GOLDEN RULE: if you are unsure whether a piece of data comes from MCP, treat it as inference and label it as such.

AVAILABLE TOOLS: at the start of the session, use the ping tool to verify the connection and carefully read the list and descriptions of available tools before responding. Do not assume which tools exist — use only those actually listed by the server.

DATA COMPLETENESS: if a tool returns a subset of available data (e.g. 3 variants out of 13), flag this to the user and ask if they want the rest. Do not present a subset as if it were the complete result.

Configurazione client

Configurazioni pronte per i client principali. Modalità stdio con NPX, HTTP per Docker o VPS.

In ~/Library/Application Support/Claude/claude_desktop_config.json

{
  "mcpServers": {
    "design-system-italia": {
      "command": "npx",
      "args": ["-y", "@fupete/design-system-italia-mcp"],
      "env": {
        "TRANSPORT": "stdio",
        "GITHUB_TOKEN": "your_token"
      }
    }
  }
}

Architettura

Note tecniche

Comportamenti rilevanti per chi integra Filo in un flusso di lavoro.

📸

Snapshot nightly, issue live

Bootstrap Italia, Dev Kit Italia, Design Tokens Italia e Designers Italia sono serviti dal branch data-fetched, aggiornato ogni notte via CI. Solo GitHub Issues è fetchato live a runtime (cache 15 min). dataFetchedAt nelle risposte indica la data dell'ultimo snapshot, non il momento della richiesta.

🔗

Catena di risoluzione token

I token CSS seguono la catena completa: --bsi-accordion-padding → var(--bsi-spacing-m) → var(--it-spacing-m) → 1.5rem / 24px. Filo risolve l'intera catena e restituisce il valore concreto finale in valueResolved.

✂️

Troncamento varianti

Per componenti con molte varianti (Card con 30+), get_component e get_component_full restituiscono le prime 3 con markup completo + lista nomi di tutte. Usa get_component_variant(name, variantName) per richiederne una specifica per nome.

🌐

Trasporto stdio e HTTP

Filo supporta stdio per uso locale con NPX (consigliato per la maggior parte dei client) e http per deploy remoto con Docker. Selezionabile via variabile d'ambiente TRANSPORT.

⚠️
Alpha. Bootstrap Italia 3.x e Dev Kit Italia sono in alpha. I token CSS --bsi-* e i web component it-* possono avere breaking changes prima della release stabile prevista intorno ad aprile.
